# Multi-Angle Review - 使い方ガイド

複数サブエージェントによる多角的コードレビューをバグ0までループ実行するスキルです。

---

## 🚀 クイックスタート

### 最もシンプルな使い方

```
〇〇\skills\multi-angle-review\SKILL.md
を読んでください。

{対象ファイルパス} を多角的にレビューしてバグ0まで回してください。
```

Claudeが以下を自動実行します：

1. 規模判定 → サブエージェント数決定
2. 並行レビュー実行
3. バグ一覧を HIGH/MEDIUM/LOW で報告
4. ユーザー承認後に修正実行
5. 再レビュー → バグ0まで繰り返し

---

## 📝 詳細な使い方

### 特定のLayerに絞る

```
backtest.py を Layer 3（ファイル間影響度）を重点的にレビューしてください。
```

### 修正しないでレビューのみ

```
レビューだけして、バグ一覧を出してください。修正はまだしないで。
```

### 特定のバグレベルのみ

```
HIGHとMEDIUMのバグのみ報告してください。LOWは不要です。
```

### 部分的な修正

```
バグ一覧の1番と3番のみ修正してください。
```

**ユーザー:**

- Agent 2体起動（ロジック + テストカバレッジ）
- バグ3件検出 → 承認 → 修正 → ラウンド2で0件確認 → 終了

### 例2: 複数ファイルのデグレ検証

**ユーザー:**

**Claude:**
- 規模判定（中〜大、スコア100）

- 3ラウンド実行でバグ0達成

## ⚠️ よくある質問

### Q. どのラウンドで終了すべきですか？

A. **バグ0 または LOWのみ残った時点**が目安です。HIGHとMEDIUMが0件なら、LOWは許容可能なケースが多いです。

### Q. 何ラウンドまで回すのが一般的ですか？

A. 以下を目安にしてください：
- シンプルな単一ファイル: 1-2ラウンド
- 複数ファイル・依存あり: 2-3ラウンド
- 大規模・モジュール横断: 3ラウンド以上

### Q. 言語指定は必要ですか？

A. **不要**です。Claudeがファイル拡張子・内容から自動判定します。言語固有のベストプラクティスはサブエージェント側が適用します。

### Q. テストがないプロジェクトでも使えますか？

A. **使えます**。テストカバレッジ担当エージェントは起動しないか、「テスト追加の提案」役に変わります。

### Q. ユーザー承認を毎ラウンド取るのは面倒ですが、自動化できますか？

A. **できません**。ソースコード編集は必ずユーザー承認制です（claude-fundamental-rulesに準拠）。ただし「全て承認」と一括承認は可能です。

### Q. 既存のコードレビュースキル（code-review, simplify）との違いは？

A. 役割分担は以下の通り：
- **multi-angle-review**: バグ検出・根絶（今回作成）
- **simplify**: コード簡素化・可読性向上
- **code-review**: PR単位のレビュー
