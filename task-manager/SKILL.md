# タスクマネージャー

このスキルは**複数のタスクセット（プロジェクト）を作成し、タスクを順番に管理・実行・完了報告する**スキルです。
ユーザーの許可を得ながら順次タスクを進め、すべて完了すると終了します。

**ファイル永続化機能**により、会話をまたいでもタスクセットが保持されます。

---

## 🎯 このスキルの目的

1. **複数プロジェクトの並行管理**
   - 独立したタスクセットを複数保持
   - 必要に応じて切り替え・再開

2. **ユーザー主導の進行**
   - 必ずユーザーの許可を得てから次へ進む
   - 勝手に進行しない安全設計

3. **柔軟なタスク操作**
   - タスクの追加・削除・変更に対応
   - 進捗状況の可視化

4. **ファイル永続化**
   - JSONファイルでタスクセットを保存
   - 会話をまたいでも継続管理可能

## 📊 データ構造

各タスクセットはJSONファイルとして保存されます：

**ファイル名:** `{セット名のスラッグ}.json`

```json
{
  "name": "デザイン作成タスク",
  "slug": "design-tasks",
  "createdAt": "2025-12-10T10:00:00",
  "updatedAt": "2025-12-10T15:30:00",
  "currentIndex": 2,
  "completed": false,
  "tasks": [
    { "id": 1, "name": "ワイヤーフレーム作成", "done": true, "completedAt": "2025-12-10T11:00:00" },
    { "id": 2, "name": "トップページデザイン", "done": true, "completedAt": "2025-12-10T14:00:00" },

    { "id": 4, "name": "レスポンシブ対応", "done": false, "completedAt": null }
  ]
}
```

### ④ タスク実行ループ

以下をタスクが尽きるまで繰り返します：

#### 4-1. 実行許可を求める

```
┏━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━┓
┃ ▶ 次のタスク                            ┃
┣━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━┫
┃                                          ┃
┃ 次のタスク「{タスク名}」を               ┃
┃ 進めてもよいですか？                     ┃
┃                                          ┃
┃ → はい / OK / 実行 で進行                ┃
┃ → スキップ で次へ                        ┃
┃ → 中断 で一時停止                        ┃
┃                                          ┃
┗━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━┛
```

#### 4-2. ユーザーが許可したら完了扱い

「はい」「OK」「実行」「進めて」などの肯定的な回答で完了とみなします。

#### 4-3. 完了報告 & ファイル更新

```
┏━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━┓
┃ ✅ タスク完了                            ┃
┣━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━┫
┃                                          ┃
┃ タスク「{タスク名}」が完了しました！     ┃
┃                                          ┃
┃ 残りのタスクは {N} 件です。              ┃
┃ 💾 進捗を保存しました。                  ┃
┃                                          ┃
┗━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━┛
```

**Claudeの処理:**
1. JSONファイルを読み込み
2. 該当タスクの `done: true`、`completedAt` を更新
3. `currentIndex` を更新
4. `updatedAt` を更新
5. JSONファイルを保存

#### 4-4. 次のタスクへ進む

自動的に次のタスクの実行許可を求めます。

**【ユーザー】**

**【Claude】**
```

┃ 管理したいタスク（プロジェクト名）を     ┃
┃ 教えてください。                         ┃
```

**【ユーザー】**

## ⚠️ 重要な注意事項

1. **ファイル永続化**
   - タスクセットは `task-sets/` にJSONファイルとして保存
   - 会話をまたいでも継続可能
   - ファイルを手動で編集することも可能

2. **許可制の徹底**
   - Claudeは勝手にタスクを進めません
   - 必ずユーザーの「はい」「OK」等の許可を待ちます

3. **柔軟な対応**
   - 「スキップ」で特定タスクを飛ばせます
   - 「中断」で一時停止できます
   - いつでもタスクの追加・削除・変更が可能です

4. **バックアップ推奨**
   - 重要なタスクセットはバックアップを取ることを推奨
   - `task-sets/` フォルダごとコピーすればOK
