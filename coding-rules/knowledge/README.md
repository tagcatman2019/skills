# Knowledge ディレクトリ

このディレクトリには、プロジェクトのコーディングルールをナレッジとして蓄積します。

## ナレッジファイル一覧

### 1. environment.md - 開発環境ナレッジ
**記録内容:**
- ビルドツール設定
- タスク構成
- ファイル監視ルール
- プロジェクト構造

**読み込み依頼例:**
- 「環境ナレッジを読んで」
- 「開発環境を確認して」

### 2. scss-rules.md - SCSSルールナレッジ
**記録内容:**
- ディレクトリ構造
- ネーミング規則
- 変数管理
- フォントサイズ単位
- ブレイクポイント

**読み込み依頼例:**
- 「SCSSルールを読んで」
- 「CSSのルールを確認して」

### 3. js-rules.md - JavaScriptルールナレッジ
**記録内容:**
- モジュール構造
- 使用ライブラリ
- イベント管理
- コーディング規約

**読み込み依頼例:**
- 「JSルールを読んで」
- 「JavaScriptのルールを確認して」

### 4. ejs-rules.md - EJSルールナレッジ
**記録内容:**
- テンプレート構造
- パーシャル分割
- コンポーネント管理
- データ受け渡し

**読み込み依頼例:**
- 「EJSルールを読んで」
- 「テンプレートのルールを確認して」

## 使い方

### ナレッジの読み込み
```
User: SCSSルールを読んでください
Claude: ✅ scss-rules.md を読み込みました
```

### ナレッジへの登録
```
User: フォントサイズはPC=rem、SP=vwです。SCSSルールに登録して
Claude: ✅ scss-rules.md に登録しました
```

### 登録確認
```
Claude: 【ナレッジ登録の確認】
この情報をSCSSルールに登録しなくていいですか？
```

## 重要な原則

1. **自動読み込みしない** - ユーザーの依頼時のみ読み込む
2. **登録は確認を取る** - 「ナレッジに登録しなくていいですか？」
3. **登録時は報告する** - どのナレッジに何を登録したか明確に
