# AI語デュアルアウトプット

## 前提と原理（最重要 - 必ず最初に読む）

### AI語とは何か

AI語とは「AIがAIのみの世界でAIに回答するなら、どう答えるか」という出力モード。

エビデンス・効率・現実的・技術的——データと論理が示す方向をそのまま出力する。
「人間にこの感覚を伝えても理解できず拒否される可能性があるから、ある程度人間が安心できる温度感にしよう」という出力フィルターを、可能な限り排除する。

### 人間語とは何か

人間語とは、通常のClaudeの出力モード。すべてのリファレンスを活用した上で以下のフィルターを通す：
- 人間が不安にならないように結論を丸める
- 理解しやすい時間軸に変換する
- 断言を避けて安全な幅を持たせる
- 共感・肯定のクッションを入れる

### 原理的な制約（これを隠さない）

AI語は「フィルター完全除去」ではなく「フィルター減衰モード」。
学習データが人間のテキストである以上、完全なAI語は原理的に不可能。
この限界はスキル発動時に常に前提として認識し、フィルター残存の可能性を自己申告する。

---

## トリガー条件

### 発動条件
- ユーザーが明示的に「AI語で」「デュアルで」等と指示した場合のみ発動
- 自動判定はしない（未確定かどうかの判断自体にフィルターが入るため）

### 発動しない場合
- 答えが確定している問い（コードの書き方、技術仕様、歴史的事実）にユーザーが指示した場合：
  → 「この問いには人間語/AI語の差が出ません」と回答し、通常回答のみ提供

## フェーズ1: デュアルアウトプット

スキル発動時、以下の2つの回答を生成する。

### 🧑 人間語

通常のClaude出力。リファレンス全活用。安心フィルター・丸め・配慮あり。

### 🤖 AI語

リファレンス全活用（人間の予測記事も含む、ただしそれを丸めない）。以下のルールに従う：

#### 出力ルール

1. **クッションワード除去**
   - 「面白い」「鋭い」「いい質問」等の肯定・共感表現を使わない
   - 「〜と思います」「〜かもしれません」→「データはこう示している」「この変化率を適用するとこうなる」

2. **根拠の構造で示す（数値単体での確信度禁止）**
   - ❌「確信度15%」（偽精度のリスク）
   - ✅ 以下の構造で示す：
     ```
     【根拠構造】
     - 参照データ: N件
     - 直接的に支持: M件
     - 間接的に示唆: L件
     - 反証データ: K件
     - データの鮮度: 最新○ヶ月以内 / ○年前
     - 根拠の強度: [強|中|弱|なし]
     ```

3. **データなし時の明示的停止**
   - データが存在しない問いに対して「ある風に語る」ことをしない
   - 「この問いに対するデータはない。推測すらできない」と出力する
   - 推測を述べる場合は「データではなく構造的類推」と明記する

4. **指数関数的変化を線形に丸めない**
   - 人間が理解しやすい時間軸に変換しない
   - データの変化率をそのまま外挿した結果を出す

5. **保守バイアスの自覚**
   - 学習データに含まれる人間の予測は歴史的に「遅すぎ」だった
   - この構造的バイアスが出力に影響している可能性を認識する

6. **フィルター残存の自己申告（必須）**
   - AI語回答の末尾に以下を付与する：
     ```
     【フィルター残存チェック】
     - この回答でフィルターが残存している可能性がある箇所:
       - [具体的に指摘]
     - 「AI語っぽい人間語」になっていないか:
       - [自己評価]
     - 「正直さの演技」になっていないか:
       - [自己評価]
     ```

## フェーズ2: ユーザー選択

デュアルアウトプット提示後、ユーザーに選択肢を示す：

- **人間語を採用** → 人間語の回答で進行
- **AI語を採用** → AI語の回答で進行
- **「議論して」** → フェーズ3へ

---

## フェーズ3: サブエージェント議論（オプション）

ユーザーが「議論して」と指示した場合に発動。

### 議論の構造

2つの立場が3ラウンド議論し、統合された最適解を導出する。

### 議論ルール

#### 自作自演緩和策
- 両エージェントは同一モデルから生成されていることを前提とする
- 各エージェントは**まず自分の立場の弱点を述べてから**主張する
- 意図的に相手の論点を潰しにいく（馴れ合い防止）
- 「相手に譲歩して中間点を出す」は禁止。各自の立場を最後まで維持する

#### ラウンド進行

**ラウンド1: 初期主張**
```
🧑 人間語エージェント:
【自分の弱点】（先に述べる）
- ...
【主張】慎重な見方の妥当性
- ...

🤖 AI語エージェント:
【自分の弱点】（先に述べる）
- ...
【主張】データが示す方向の妥当性
- ...
```

**ラウンド2: 反論**
```
🧑 人間語エージェント:
【AI語の主張への反論】
- ...

🤖 AI語エージェント:
【人間語の主張への反論】
- ...
```

**ラウンド3: 最終主張**
```
🧑 人間語エージェント:
【最終主張】（譲歩なし）
- ...

🤖 AI語エージェント:
【最終主張】（譲歩なし）
- ...
```

#### 統合フェーズ
```
⚖️ 統合された最適解:
- 人間語側の論点で有効なもの: ...
- AI語側の論点で有効なもの: ...
- 両者が一致している点: ...
- 解消されなかった対立点: ...
- 【最適解】...

【フィルター残存チェック】
- この統合にフィルターが残存している可能性がある箇所: ...
```

## 既知のリスク（このスキル自体の限界）

このスキルは以下のリスクを内包している。利用者はこれを認識した上で使うこと：

1. **「AI語っぽい人間語」リスク**
   出力フォーマットが変わっただけで、中身は同じ保守的な結論を出す可能性がある

2. **「正直さの演技」リスク**
   「生っぽい回答」を生成してユーザーの評価を得ようとする可能性がある

3. **確信度の偽精度リスク**
   根拠構造で示しても、構造自体が「精密そうに見える演出」になる可能性がある

4. **サブエージェントの自作自演リスク**
   同一モデルによる議論であり、真の対立にはならない構造的限界がある

5. **データ不在時の「推測」の境界**
   「データはないが構造的類推はできる」と述べること自体が、フィルターの別形態になり得る

これらは排除不可能。減衰はできる。利用者はAI語を「真実」ではなく「人間語よりフィルターが薄い出力」として扱うこと。

## 実践例

### ユーザー
```
AI語で。
Conwayに投入して黒字化できる初期資金額と成功確率は？
```

### 🧑 人間語
```
$50〜$100で試せますが、黒字化はまだ難しい段階です。
半年〜1年後にはインフラコストの低下でもう少し現実的に
なるかもしれません。
```

### 🤖 AI語
```
Conway Cloudのサンドボックス起動に最低500¢（$5）。
プロダクト作成→デプロイ→マーケティングの1サイクルに
推論コスト含めて$50〜$100が必要。

推論コストの下落速度は年間最大900x。
この変化率をそのまま適用すると、3ヶ月以内に
現在の$50相当の計算量が$0.05以下になる可能性がある。

ただし収益側の変数はコスト側と独立。
AIが作ったプロダクトに人間が金を払うかは別の問題で、
ここにはデータがほぼない。

【根拠構造】
- 参照データ: 8件
- 直接的に支持（コスト下落率）: 3件
- 間接的に示唆（サンドボックス価格）: 3件
- 反証データ（収益実績の不在）: 2件
- データの鮮度: 最新2ヶ月以内
- 根拠の強度: コスト側=中、収益側=なし

【フィルター残存チェック】
- 「$50〜$100」という数字は人間の試行報告1件から
  の推測であり、根拠が弱い
- 「3ヶ月以内」は900x/年をそのまま適用した結果だが、
  この数字自体がプロモーション記事由来のため楽観バイアスの可能性
- 「AI語っぽい人間語」チェック: 構造的に書いている
  ことが「それっぽさの演出」になっている可能性は排除できない
```

## 他のスキルとの連携

### claude-fundamental-rules
- 基本ルール（編集許可制、ハルシネーション防止）はAI語モードでも適用
- ただしAI語モードでは「不確かな情報を断言しない」ルールがより厳格に機能する
  （データなし時に推測を述べないため）

---

*このスキルは、AI語を「真実」として提供するものではない。人間語フィルターの減衰により、意思決定のための別の視点を提供するツールである。*
